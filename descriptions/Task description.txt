Решаем обратную задачу спектроскопии: 
По 2D-картам фотолюминесценции (ФЛ) определяем концентрации ионов металлов в среде. 
Данный эксперимент исследует эффективность разложения 2D-карт ФЛ по базису Лежандра, Фурье и Цернике для получения вектора фичей, на которых в дальнейшем обучаются модели. По идее, это должно решать проблему коррелированности данных в картах (базисы - ортогональные); эффективно сжимать карты с ~7000 признаками в ~100 признаков, что важно при ограниченности выборки (~7000 примеров).


Дизайн эксперимента:
Сравниваем метрику MAE (можно дополнительно RMSE и R2) при использовании разных базисов разложения с разным порядком разложения и на разных моделях.

Базисы разложения (они же набор моментов):
1. MomentumTransformer2D с дефолтными параметрами кроме order (x_bounds и y_bounds не меняй, а то плохо начинает работать),
2. LegendreTransformer2D с дефолтными параметрами кроме order,
3. FourierTransformer2D с дефолтными параметрами кроме order,
4. ZernikeTransformer2D указывай x_bounds, y_bounds наших карт ФЛ (те что в нанометрах), остальные дефолтные

Порядки разложения (order):
order определяет порядки разложения, однако для разных моментов это даст разное количество фичей на выходе после преобразования (n_features). Для начала проведем исследование для n_features в примерном диапозоне от 1 до 100. Для методов MomentumTransformer2D, LegendreTransformer2D и LegendreTransformer2D это достигается варьирование order in range(0, 13+1), при order = 13 n_features = 105. Для метода FourierTransformer2D order in range(0, 7+1), при order = 7 n_features = 113.

Набор моделей: 
1. Линейная регрессия из sci-kit learn (LR), 
2. Случайный лес из sci-kit learn (RF), 
3. Градиентный бустинг из xgboost (GB),
4. Многослойный персептрон (MLP). 
Везде установи одинаковый validation_train_ratio = 2 / 7 (тот, что используется для ранней остановки обучения), по возможности. 


Пайплан выглядит так:
1. Нормировка данных карт ФЛ (на всем наборе высчитывается максимум ФЛ, на него нормируются все спектры - в тренировочном и тестовом наборах).
2. Использование одного из методов агрегации карт ФЛ.
3. Применение метода машинного обучения.
4. Валидация на тестовом наборе, получение метрик. 

Нормировка y - MinMaxScaler, раздельно на тренировочном и тестовом наборах.

Обучение методом кросс-валидации CV на 10 фолдах.

Бейзлайн: сверточная нейронная сеть с аналогичными нормировками X и Y и CV. Нужно обернуть ее применение в класс CNNRegressor(RegressorMixin, BaseEstimator), определив методы __init__, fit, predict, чтобы пользоваться схожим пайплайном с основными методами. CNNRegressor должен встраиваться на место 2 и 3 пунктов из пайплайна.
Приблизительная архитектура CNN (она же указана в ноутбуке Baseline.ipynb . Возьми от туда только архитектуру и общие черты обучения. Полностью списывать алгоритм обучения не надо, он плохой. Используй стандартные подходы с torch):
class twoD_CNN_new_leakyrelu_32(nn.Module):
    
    def __init__(self):
        
        super().__init__()

        self.CNN = nn.Sequential(
            nn.Conv2d(in_channels=1, out_channels=3, kernel_size=5, stride=5),
            nn.LeakyReLU(),
            nn.MaxPool2d(2), 
            nn.Flatten(),
            nn.Linear(3 * 20 * 2, 32), 
            nn.LeakyReLU(), 
            nn.Linear(32, 7) 

        )

    def forward(self, x):
        x = self.CNN(x)
        return x 



Анализ результатов:
Для результатов на каждой модели строим свой график (всего, получается, 4 графика).
На каждом графике строим 7 осей в столбик - по количеству ионов.
Оси X графиков - количество входных признаков n_features, которое генерирует тот или иной базис разложения. Эти оси должны быть синхронизированы между всеми 7-ю графиками.
Оси Y графиков - средние метрики MAE с погрешностями, полученные в результате CV. Рисуй их как точки с вертикальными воротами, соединенные линией для точек с одним типом набора базисных моментов. 
Маркеры и тип соединяющей линии - определяется в легенде (4 типа, по количеству базисов разложения).
Результаты метрик baseline нарисовать как серую горизонтальную линию с прозрачной закраской области +-std.
